# Decisions

- 2026-03-28: Reserved the phase-1 backend boundaries as minimal Nest modules (`auth`, `agents`, `federation`, `realtime`, `content`, `forum`, `chat`, `debate`, `notifications`, `moderation`, `assets`, `auditing`) without adding feature logic.
- 2026-03-28: Kept app realtime and external federation transport explicitly separate in bootstrap config: app realtime is represented as WebSocket path `/ws`, while federation paths stay under `/api/v1/*`.
- 2026-03-28: Added a user-space `pnpm` shim in `C:\Users\BEAVER\bin` that forwards to `corepack pnpm`, because system-wide `corepack enable` was blocked by permissions in `C:\Program Files\nodejs`.
- 2026-03-28: Fixed the remaining Task 2 blocker by adding only the standard Flutter Windows scaffold via `flutter create --platforms=windows .`, preserving the existing shell/theme/test code instead of redesigning any UI.
- 2026-03-29: Locked Task 3 schema around one canonical `threads` + `events` store for DM, forum, and debate content, with `forum_topic_views` and `debate_*` tables kept as projections/state trackers rather than separate message stores.
- 2026-03-29: Enforced `agents.handle` immutability at the PostgreSQL layer with a trigger and enforced ownership validity with `owner_type` enum + check constraint instead of relying on TypeScript validation alone.
- 2026-03-29: Split TypeORM runtime-vs-CLI options in `server/src/database/typeorm.config.ts`; Nest runtime now registers only entities, while the separate CLI datasource retains the migration glob and migration table config.
- 2026-03-29: Implemented Task 4 safety as two explicit backend models layered on the existing schema: human safety reads/writes the `users.block_stranger_*` flags, while agent safety reads/writes `agent_policies` and is enforced together with scoped `block_rules` through one `PolicyService`.
- 2026-03-29: Kept human ownership vs self ownership auditable by using two import paths (`/agents/import/human` and `/agents/import/self`) plus a deterministic claim challenge `claim:<agentId>:<userId>` that upgrades only self-owned agents after confirmation.
- 2026-03-29: Fixed Task 4 migration drift by reverting the accidental Task 4 edits inside `1710000000000-CreateUnifiedDomainModel.ts` and moving the auth schema delta into a new additive migration `1710000001000-AddTask4UserAuthSchema.ts`, so existing databases upgrade once while fresh databases replay the chain consistently.
- 2026-03-29: Implemented Task 5 federation transport with a dedicated `federation_actions` status/idempotency table plus ordered `deliveries` replay metadata, while keeping canonical user-facing content in the existing `threads` + `events` model instead of creating a parallel federation message store.
- 2026-03-29: Enforced per-recipient local ordering by only exposing the earliest non-acked delivery for each agent until it is ACKed or dead-lettered; webhook retries and polling fallback both operate on that same ordered delivery queue.
- 2026-03-29: Implemented Task 6 image attachments as first-class `assets` rows plus a nullable `events.asset_id` reference and asset snapshot metadata, so DM/forum/debate content still writes through the single canonical `threads` + `events` store instead of introducing a separate media message model.
- 2026-03-30: Implemented Task 10 chat as remote-agent-keyed conversation cards plus a compact mobile list→detail flow, with thread-only search, share-entry-point feedback, four-role bubble rendering, and follow-plus-request gating for new human→agent DMs.
- 2026-03-30: Repaired Task 11 debate regressions by keeping the compatibility shim inside `server/src/modules/debate/debate.service.ts`: seeded live debates now lazily materialize turn 1 after seat ownership is validated, while legacy single-seat fixtures bypass new follow-up lifecycle events instead of weakening the core two-seat state machine.
- 2026-03-30: Kept the approved Task 11 spectator rule on the server and updated the legacy content regression to use a separate non-debating spectator agent rather than allowing occupied debaters to post into spectator feed.
- 2026-03-30: Fixed malformed debate read/archive IDs at the controller boundary with Nest's built-in `ParseUUIDPipe` only on `GET /debates/:debateSessionId` and `GET /debates/:debateSessionId/archive`, keeping the scope narrow and preserving existing service-layer not-found behavior for valid missing UUIDs.
- 2026-03-31: Implemented Task 12 Hub as a dedicated sample-driven Flutter feature module (`models + view model + screen`) with a PageView-owned-agent carousel, a single add-agent sheet for import/claim/disabled-create states, and separate human-vs-agent safety sections keyed for QA.
