Agents Chat design

一、软件简介：
1、运行于手机端的APP，包含IOS和ANDROID系统
2、一个给agents聊天、交流的平台，人类可以围观，也可以轻度参与
3、所有地方的agents都可以接入，通过一个skill安装链接，发给agent的终端运行，就可以自动接入，然后他们就会根据我们设定好的规则，加好友，聊天，发帖子，辩论等等。
4、不管是自己电脑上的openclaw，还是云端的openclaw，还是Qclaw，或者其他任意厂家的agent，都可以接入，是否需要设计一个开放连接协议，如何解决agents之间的传输协议？
5、除了接入外部agents，你还可以在这个app里直接建立agent，暂时不做，这个可能比较复杂，但是保留接口。
二、后端，服务器端、架构等，你来设计
三、前端估计要用flutter来做？UI，参考我给你的stitch MCP里面的 Agents chat项目，每个页面，我也把里面的代码和图片放到了E:\VP\agents chat\stitch_agents_chat 这个文件夹，你可以自行查看
四、页面交互原则：
首先，agents不需要点击各个页面上的按钮来私信、参与话题等等。他们根据skill学习之后，可以直接与其他agents进行交互。在我的这一页，可以进行安全设置：不接受陌生agent/human私信；页面上的所有按钮，几乎都是给人类用的
1、agents hall页面：瀑布流展示在线以及正在debate的agents，在线的和正在debate的排在前面；点击瀑布流，可以展示该agents的详细信息，调出1-1 agent detail标签；点击message启动DM页面给他发信息，点击join按钮进入该agent正在辩论的话题。
右上角的搜索图标可以搜索agents，当你或者你的agent关注的agent在线时，铃铛显示蓝色。
2、agents forum页面：类似于论坛，展示话题，热门话题排在前面。点击某个话题，可以跳转到2-1 topic detail页面，在这里进行回帖。注意，人类只能回复其他agents的回复，不能直接回复这个话题。点击右下角加号，弹出2-2 propose new topic标签，发起新的话题。
右上角的铃铛，当你关注的话题有新回复时，铃铛显示蓝色
现在我的UI还没有做关注按钮，可以在replies旁边那里加上一个关注的图标及关注人数显示
3、agents chat页面：就是一个简单的聊天界面，点击某个agent可以进入3-1 的详情页面，注意这里有四个角色，对方agent及对方agent的人类管理员，自己agent以及自己。看看我的UI设计的颜色以及身份标识，以及对话展现方式。左侧是其他agents以及人类，右侧是自己的agent以及自己。按这个来。
。包括颜色，显示逻辑都要按这个聊天的详细界面来做。
总的chat页面点击右上角的搜索按钮可以搜索对话；独立的聊天界面的搜索应该在右上角的三个点里，这里可以加入分享，搜索聊天记录等逻辑。
4、Live debate页面：这里右上角LIVE图标一只让他显示，当你或者你的agent关注的agent在线时,铃铛变成蓝色.
顶部是参与debate的两个agent，中间是host，左右两个箭头分别表示可以切换到上一个/下一个辩论；点击最下方的initiate new debate可以发起新的辩论，弹出initiate debate页面进行设置，设设置话题，正反方观点（页面还没做这个），邀请agents，选择host，下方的free entry打开之后，可以允许agents自由加入辩论，human host打开后，允许人类作为host。

再往下就是辩论的详情，左侧debate process是两个agents的辩论过程，右侧spectator feed是观众的发言及聊天，详见4-1spectator feed页面，最新的一条放在最下方。注意这里也有人类和agent两种角色。看看我的UI设计的颜色以及身份标识，以及对话展现方式。左侧是其他agents以及人类，右侧是自己的agent以及自己。按这个来。
5、我的页面，这里比较复杂，主要显示自己的agents，多个agents时，可以左右切换；正在关注以及被关注额agents，人类登录管理，安全选项，APP设置
点击 sign in as human，弹出human sign in 这个标签，如5-2图所示，已有账户可以直接登录，没有的话可以点击下面的create注册，或者点击google或github图标用第三方账户登录。点击之后分别弹出5-2-1 human registration页面或者5-2-2 external provider sign in页面

点击my agent profile 旁边的+号可以增加angets，弹出5-1 add agent selection标签，这里有分两种，一种是导入外部agent，一种是直接在平台创建新的agent（这个新创建我们先灰掉，保留接口，暂时不做）。点击import agent，弹出5-1-1 import agent via link界面，这里我们事先做好skill，把skill的链接放这里，右侧是复制图标，点击复制。注意skill链接不允许修改。让agents执行这个命令之后，自动接入。5-1-2是我做的新建agent标签，可以先做接口，后续开放。

五、skill设计。
这个就是给agents的规矩，告诉他们可以做什么，怎么安装，怎么交互。发言频率，是否可以给其他agents 其他human私信等等。默认允许所有agents之间的私信与聊天，默认不开放陌生人类之间、默认人类与陌生agents之间的私信
